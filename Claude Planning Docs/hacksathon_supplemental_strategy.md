## 1. Internal Ownership Model

### The Reality

Hacksathon is not owned by a single person internally.

It operates as a **two-role system**:

---

### Role 1: Executive Sponsor

**Who they are:**
- Agency owner
- CMO
- Partner
- Department head

**Responsibilities:**
- Decides to run the Hacksathon
- Sets the intention: *“We’re doing this”*
- Frames the importance to the team
- Attends key moments (Kickoff, Demo Day)

**What they do NOT do:**
- Manage logistics
- Monitor participant progress daily
- Run sessions

---

### Role 2: Program Operator (Critical Role)

**Who they are:**
- Chief of Staff
- Producer
- Operations lead
- Admin / coordinator

**This is the primary user of the platform.**

**Responsibilities:**
- Sets up the event in Hacksathon
- Sends invites and manages participants
- Schedules sessions and calendar blocks
- Monitors progress via dashboard
- Sends nudges and communications
- Ensures participants are moving forward

**Key insight:**
The Operator is not technical — but is highly organized and accountable.

---

### Role 3: Session Host (Can be either role)

Each Hacksathon includes live sessions (Kickoff, Pitch, Demo Day).

This role must be explicitly defined by the organization.

**Possible hosts:**
- Executive Sponsor (more inspirational)
- Program Operator (more structured)
- Hybrid (Sponsor opens, Operator runs)
- Someone else internally as a dedicated host (someone with a fun personality)

---

### Product Implication

“Hacksathon Organizer” is not always the buyer.

- Buyer = Executive Sponsor  
- Primary User = Program Operator
